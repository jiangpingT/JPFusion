# ADR-0003: Lawson 准则驱动奖励 + 防 gaming 设计

**状态**：接受

**日期**：2026-02

**决策人**：姜哥 / 阿策

---

## 背景

FusionEnv 的奖励函数设计是整个 Phase 1 最关键的决策。

从 JPRobot 后空翻的经验（V25~V49）来看，gaming 是所有 RL 项目的核心挑战。
已知的 gaming 规律：
1. **任何 per-step 正奖励都是连续收入流**：Agent 宁可在低分状态驻留刷每步正奖励，也不愿冒险完成高难度动作（JPRobot V38-V40 教训）
2. **一次性大奖励才能引导真实行为**：JPRobot V41 引入 `W_UPRIGHTNESS_BONUS = 0.5`，一次性直立加成，效果显著
3. **功率暴涨 gaming**：Agent 可能把 P_heat 拉满 → T_e 飙升 → Lawson 参数瞬间爆高 → 但同时 β_N 超限破裂（这是等离子体物理的真实约束）
4. **卡边 gaming**：Agent 在 q95 = 2.01 游荡，不破裂但不稳定

需要设计一个奖励函数：
- 有科学意义（驱动真实物理目标）
- 防止各种 gaming 模式

## 决策

采用以下奖励结构（**无 per-step 正奖励**）：

```python
total_reward = Lawson 对数奖励（可正可负，基准为 0）
             + 效率惩罚（-0.05 × P_heat_MW，每步，负值）
             + 稳定性惩罚（接近破裂边界时，负值，不会被 gaming）
             + 成功一次性奖励（首次跨越 Lawson 准则：+500，不重复）
             [破裂时] = -200（终止惩罚，一次性）
```

**设计原则**：
1. Lawson 对数奖励：`W_LAWSON × log10(lawson / 3e21)`——远低于目标时给强负信号，接近目标给弱正信号，对数尺度防止奖励爆炸
2. 效率惩罚：功率不是免费的，浪费功率有代价——防止功率暴涨 gaming
3. 稳定性惩罚：只在接近边界（q95 < 2.5）时施加，远离时不惩罚——不产生 per-step 正奖励
4. 一次性成功奖励：只在从低于目标跨到高于目标时计算——防止重复计算
5. 破裂终止惩罚：一次性大负奖励，明确告诉 Agent 破裂是最坏结果

## 后果

### 正面影响

- 消除"卡边"gaming（效率惩罚让 P_heat 最优化，稳定性惩罚让 q95 远离边界）
- Lawson 对数奖励提供连续梯度信号（即使远低于目标也有方向）
- Phase 1 快速测试（5000 步）验证：破裂率 0%，无 gaming
- 奖励分量可在 Dashboard 实时监控（info 字典包含所有分量）

### 负面影响 / 权衡

- 奖励稀疏：Lawson 准则门槛高（n·T·τ > 3×10²¹），初始阶段 Agent 很难触发成功奖励
- W_LAWSON = 10.0 可能需要根据实际训练曲线调整（参考 JPRobot 调参经验）
- 对数奖励在远低于目标时会产生大负值，可能导致早期训练不稳定

### 待验证的 Gaming 风险

| 风险 | 监控指标 | 应对 |
|------|----------|------|
| 功率暴涨破裂 | `P_heat_norm > 0.9` 的步数比例 | 增大效率惩罚权重 |
| q95 卡边 | `q95_margin < 0.5` 的步数比例 | 增大稳定性惩罚权重 |
| 低密度策略 | `n_e_norm < 0.1` 长期驻留 | 增加密度下限惩罚 |

## 替代方案

### 方案 A：形状奖励（Reward Shaping）——每步都给 Lawson 参数正比奖励

**放弃原因**：产生 per-step 正奖励流，Agent 会在任何 Lawson 值大于 0 的状态反复驻留，
不会探索更优策略。这正是 JPRobot V38-V40 的失败模式。

### 方案 B：终止奖励（Episode 结束时计算总 Lawson 积分）

**放弃原因**：奖励完全稀疏，credit assignment 困难，长 episode（500 步）情况下
初始探索几乎不可能收敛。

### 方案 C：势函数奖励（Potential-based Shaping，理论上不改变最优策略）

**放弃原因**：需要定义合适的势函数，等离子体物理中势函数的物理含义不直观，
调试困难。未来可以作为研究方向。
